What We Built

A fully local AI assistant

DashAI runs large language models directly inside your browser tab using the WebGPU API. When you send a message, the response is generated entirely on your own GPU — no request leaves your device, no API is called, no server exists.

It's the same technology that powers native AI applications, brought into the browser through open-source work by the WebLLM project and quantised open-weight models from Meta AI.

lock_open DashAI is built entirely on open-source foundations — WebLLM, React, Prism.js, and open-weight AI models released under public licences.

Technology

How DashAI works

public
You open DashAI in your browser
The page loads static HTML, CSS, and JavaScript files. No account, no login, no tracking pixel. Google Fonts, Prism.js, and React are loaded from CDNs.
download
You choose an open-weight model
The model weights (e.g. Llama 3.2 1B, ~700 MB) are downloaded from jsDelivr CDN or Hugging Face directly to your browser's cache. This happens once — subsequent visits use the cached version. Hugging Face may log standard CDN access data (IP, browser) for this download.
memory
WebLLM loads the model onto your GPU
WebLLM — an open-source library from the MLC AI project — uses the WebGPU API to load the quantised model weights into your GPU's VRAM. This is the same approach used by native AI desktop apps, adapted for the browser.
chat
You chat — entirely offline
Every token of every response is generated locally on your hardware. No network request is made. You can now disconnect your internet entirely and DashAI keeps working.
delete
Tab closed → all data gone
Conversation history lives only in browser RAM. When you close the tab, it's gone. The only thing that persists is your model choice in localStorage — one string, on your device only.
